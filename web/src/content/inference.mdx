Your agent needs a **brain** — somewhere to send its thinking. There's no GPU on this box, so we use a **remote, OpenAI-compatible inference endpoint**. You provide three things:

- **Base URL** — the endpoint, ending in `/v1` (e.g. `https://integrate.api.nvidia.com/v1`)
- **Model** — the model id to use (e.g. `nvidia/llama-3.3-nemotron-super-49b-v1`)
- **API key** — your provider key (e.g. an `nvapi-…` key from [build.nvidia.com](https://build.nvidia.com))

<Callout type="note" title="The privacy router — why your key never enters the agent">
We don't paste the key into the agent. Instead we register the endpoint + key with the <strong>gateway</strong> as a <em>provider</em>, then point the gateway's inference route at it. Your agent later sends its thinking to a virtual host, <code>https://inference.local</code>; the supervisor intercepts that <em>before</em> policy is even evaluated and routes it to the gateway, which injects the real key and calls upstream. So the agent <strong>never holds your key</strong>, and the real inference host <strong>never needs an entry in the agent's egress policy</strong>. That's the OpenShell privacy router.
</Callout>

## Step 1 — Put your values in `.env`

Open the launchable's `.env` in an editor:

```bash
cd ~/nemoclaw-openshift-launchable
nano .env        # or: vi .env
```

Set these three lines (replace the values with your own):

```bash
NEMOCLAW_INFERENCE_BASE_URL=https://integrate.api.nvidia.com/v1
NEMOCLAW_MODEL=nvidia/llama-3.3-nemotron-super-49b-v1
NEMOCLAW_API_KEY=nvapi-your-key-here
```

Save and exit (in `nano`: <kbd>Ctrl-O</kbd>, <kbd>Enter</kbd>, <kbd>Ctrl-X</kbd>).

## Step 2 — Load them into your shell

```bash
cd ~/nemoclaw-openshift-launchable
set -a; . .env; set +a
echo "URL=$NEMOCLAW_INFERENCE_BASE_URL  MODEL=$NEMOCLAW_MODEL  KEY=${NEMOCLAW_API_KEY:0:6}…"
```

You should see your URL and model, and the key's first few characters (we don't print the whole key).

## Step 3 — Verify the endpoint actually works

Before registering it, sanity-check the endpoint with the standard OpenAI-compatible `/v1/models` call, authenticated with your key:

```bash
curl -sS "$NEMOCLAW_INFERENCE_BASE_URL/models" -H "Authorization: Bearer $NEMOCLAW_API_KEY" | head -c 600; echo
```

A JSON list of models (look for `"object": "list"` and model `"id"`s) means your **endpoint + key are good**. Confirm your chosen model is in the list:

```bash
curl -sS "$NEMOCLAW_INFERENCE_BASE_URL/models" -H "Authorization: Bearer $NEMOCLAW_API_KEY" | grep -o "\"id\":\"[^\"]*\"" | head -20
```

<Callout type="warn" title="If it fails">
<strong>401 / Unauthorized</strong> → wrong or expired key. <strong>404</strong> → the base URL probably shouldn't (or should) end in <code>/v1</code> — match your provider's docs. <strong>Connection error</strong> → check the host name. Fix <code>.env</code> and re-run Step 2 + 3 before moving on.
</Callout>

## Step 4 — Register it with the gateway as a provider

Hand the endpoint + key to the gateway. From here on *the gateway* holds them — not you, and not the agent:

```bash
openshell provider create --name default --type openai \
  --credential OPENAI_API_KEY="$NEMOCLAW_API_KEY" \
  --config OPENAI_BASE_URL="$NEMOCLAW_INFERENCE_BASE_URL"
```

The gateway probes the endpoint and prints the **validated endpoints** it found (e.g. `…/v1/chat/completions`). The key is now stored gateway-side as a credential — not in any file you'll hand the agent.

## Step 5 — Point the inference route at your model

Tell the gateway which provider + model to use when anything asks for inference:

```bash
openshell inference set --provider default --model "$NEMOCLAW_MODEL"
```

## Step 6 — Confirm the route

```bash
openshell inference get
```

You should see `Provider: default` and your `Model:`. That's the route every agent reaches via `inference.local`.

<Callout type="rh" title="Brain wired — through the router">
Endpoint verified, registered as a gateway provider, and the inference route is set. Your key lives in the gateway, not the agent. When you build the agent next, it points at <code>https://inference.local</code> — so its egress policy won't even mention your inference host. On to creating the agent.
</Callout>
