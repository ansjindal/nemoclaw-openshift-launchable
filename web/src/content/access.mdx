The stack is up. Two UIs to open — both are published as host ports and exposed via your Brev tunnels.

## Open the OpenClaw UI

It's published on host port **`30789`** — **open the `openclaw` tunnel URL straight from your Brev instance page** in a browser (no shell, no port-forward). Log in with the workshop password.

First connection will ask for **device pairing** — that's the next lesson. (If you just want to confirm it's serving before opening, `curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:30789/` in the shell returns `200`.)

## Open the OpenShift console

If phase 60 ran, the console is at host port **`30900`** under `/console/`:

```bash
curl -s -o /dev/null -w "console :30900 -> %{http_code}\n" http://127.0.0.1:30900/console/
```

<Callout type="note" title="Console shows ‘no resources’?">
Expected — MicroShift lacks the OpenShift <code>project.openshift.io</code> API the console scopes to. Switch the **Project dropdown to “All Projects”**, or browse <code>/console/k8s/all-namespaces/pods</code>.
</Callout>
