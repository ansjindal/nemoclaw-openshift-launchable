The stack is up, and both web UIs are already published as host ports — so **Brev gives you a clickable URL for each on your instance page**. Just open them; no shell, no port-forward.

## Open the OpenClaw UI

On your Brev instance page, open the tunnel for port **`30789`** in a browser. Log in with the workshop password.

First connection will ask for **device pairing** — that's the next lesson.

## Open the OpenShift console

If the console phase ran, open the Brev tunnel for port **`30900`** and go to **`/console/`**.

<Callout type="note" title="Console shows ‘no resources’?">
Expected — MicroShift lacks the OpenShift <code>project.openshift.io</code> API the console scopes to. Switch the **Project dropdown to “All Projects”**, or browse <code>/console/k8s/all-namespaces/pods</code>.
</Callout>

<Callout type="tip" title="Don't see the tunnels?">
Brev exposes a URL per published host port on your instance page. If a port isn't listed, add it there — these are plain host ports (<code>30789</code>, <code>30900</code>), no extra setup.
</Callout>
