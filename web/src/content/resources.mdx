## The toolbar (top-right of every page)

Three live tools sit in the header, next to the Red Hat / NVIDIA chips:

- **Live** → [Live OpenShell](/learn/live). Reads your gateway straight from the cluster: gateway health + version, the **sandbox fleet**, the **inference route** (active provider + model, key names only), and — click any sandbox — its **Logs / audit**, **Policy decisions** (every egress attempt the policy *allowed* or *denied*, color-coded with the host, binary, rule, engine, and denial reason), and effective **Policy** (full YAML). Auto-refreshes; served read-only via `/api/openshell`.
- **Approvals** → [/approvals](/approvals). Two human-in-the-loop inboxes, with a live pending-count badge:
  - **Access requests** — when the agent is denied reaching an API/website, the gateway proposes an allow-rule; **Approve** merges it into the agent's *live* egress policy, **Deny** rejects it (the gateway draft-policy RPCs).
  - **Device approvals** — browsers/CLIs asking to pair with the agent's Control UI; approve or deny each.
- **Links** — opens your **Brev instance's services** in a new tab:
  - **OpenClaw UI** — your agent's Control UI (chat + device pairing), at `openclaw-<id>.stg.apps.launchpad.nvidia.com`.
  - **Grafana** — the observability dashboards (same-origin `/grafana`, no ID needed).
  - **OpenShift console** — the cluster web console (`openshift-<id>.stg.apps.launchpad.nvidia.com/console/`).

  The dropdown also lets you set/override your **Brev instance ID** — it's auto-detected from this page's URL (the suffix in `…-<id>.stg.apps.launchpad.nvidia.com`); set it by hand if a link doesn't resolve.
