This page reads your **live gateway** — the same fleet `openshell sandbox list` shows, rendered right here and refreshed every few seconds. It's a quick way to glance at what's running without dropping into the shell.

## What you're looking at

- **OpenShell gateway** — green/Connected means the control plane is up (and its version). This is the gateway behind `127.0.0.1:30808`.
- **Sandboxes** — every agent the gateway is managing, by name, with its phase, readiness, restart count, and age. Your `shifty` agent shows up here once you've built it; create more in Part VI and they appear too.

## Where the data comes from

The workshop server runs on the instance, so it reads the cluster directly — no extra component, no OpenShell UI to deploy:

```bash
oc -n openshell get sandboxes.agents.x-k8s.io -o json
```

The page calls a tiny read-only API route (`/api/openshell`) that runs exactly that (plus the gateway's StatefulSet for health) and returns trimmed JSON. Because the gateway speaks gRPC and the sandboxes are plain Kubernetes objects, *any* surface can render the fleet — the `openshell` CLI, the companion `openshell-ui` console, or a panel like this one stitched into your own tooling.
