You understand the pieces — now **play with the whole control plane like an operator** before you build for real: observe it, govern it, expose it, using the `openshell` CLI. We'll create a **throwaway sandbox** just to poke at (no OpenClaw, no model — that's Part IV). By the end you'll have touched every part of OpenShell. (All of this runs in the lab shell against your live gateway.)

Spin up the throwaway sandbox. `--from` picks the sandbox image — here the OpenClaw community image (it's already pulled on the node; this lab won't actually run OpenClaw). `-- true` runs once and exits, so `create` **returns right away** while the supervisor keeps the box alive; `--no-tty` runs it non-interactively. (Pass a long-running command like `sleep 1800` instead and `create` would stay attached and block your shell.)

```bash
openshell sandbox create --name lab --no-tty \
  --from ghcr.io/nvidia/openshell-community/sandboxes/openclaw:latest \
  -- true
```

List your sandboxes:

```bash
openshell sandbox list
```

## 1. Is the gateway healthy?

```bash
openshell status
```

`Status: Connected` + a version is the control plane answering. And a deeper preflight:

```bash
openshell doctor check
```

## 2. Inspect a sandbox

```bash
openshell sandbox get lab
```

Note the **Phase**, the **Policy source** (sandbox vs gateway-global), and the **Revision** — every policy change bumps it. The effective policy is printed inline.

## 3. The audit trail — `logs`

This is OpenShell's observability: every gateway + sandbox action, structured. Watch the bootstrap you learned about actually happen:

```bash
openshell logs lab --source gateway
```

You'll see `minted sandbox JWT`, `Creating sandbox in Kubernetes`, `issued gateway sandbox JWT` — the exact handshake from the architecture lesson. Filter and stream:

```bash
openshell logs lab --source sandbox --level debug
# live tail (Ctrl-C to stop):
openshell logs lab --tail
```

<Callout type="note" title="This is the SIEM feed">
Each entry is an OCSF-style security event. In production you stream these to your SIEM — every allow/deny, every policy change, every exec, attributable to a sandbox.
</Callout>

## 4. Govern it — `policy`

See the effective policy, its history, and (the fun part) **prove** a property of it:

```bash
openshell policy get lab        # effective policy
openshell policy list lab       # revision history
```

`openshell policy` also does `set` / `update` (hot-reload egress on a running sandbox — no restart) and `prove` (formally check "can this agent ever reach host X?" — or get a counterexample). That's the audit→enforce loop from the policy lesson, live.

## 5. Where does its model come from? — `inference`

```bash
openshell inference get
```

The gateway's **privacy router**: register a backend as a *provider* (endpoint + key, held gateway-side), then `openshell inference set` points the route at a provider+model. Gateway-created sandboxes reach it through `inference.local` — so those agents never hold the upstream key, and the real host never appears in their egress policy. That's exactly how you'll wire up your own agent in Part IV.

## 6. Expose a port — `forward` / `service`

If a sandbox serves HTTP, reach it without `oc`:

```bash
openshell service list           # exposed sandbox services
openshell forward list           # active port-forwards
```

(You'll use `openshell forward start` for real in *Open the Agent UI* to expose your agent's Control UI.)

## Clean up

```bash
openshell sandbox delete lab
```
