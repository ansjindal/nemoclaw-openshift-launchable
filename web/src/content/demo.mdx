Chatting with a copilot is great for *operating* — but to **demo** what you built (a Pod Doctor, a right-sizer, an incident bot), you want a surface that shows the **task**, not a chat log. The answer: have your copilot **serve its own endpoint**, and open it at a real URL.

## The idea

Your agent runs in a sandbox — a real pod with a real network. Anything it serves on a port can be published. So:

1. Your copilot **serves an HTTP endpoint** — a status page, a `/diagnose` API, a tiny dashboard.
2. You **expose that port** to a Brev URL.
3. You **demo at the URL** — judges see your copilot *doing the thing*, live.

## Build a tiny Pod Doctor dashboard

If you finished *Extending Shifty*, you already have a `pod-doctor` sandbox with a read-only Kubernetes token at `/workspace/cluster/token`. Drop in a tiny server that uses that credential and serves a live dashboard on port `8000`:

```bash
oc -n openshell exec -i pod-doctor -c agent -- sh -c 'cat > /workspace/pod-doctor-demo.js && nohup node /workspace/pod-doctor-demo.js >/workspace/pod-doctor-demo.log 2>&1 &' <<'JS'
const http = require("http");
const https = require("https");
const fs = require("fs");

const token = fs.readFileSync("/workspace/cluster/token", "utf8").trim();
const ca = fs.readFileSync("/workspace/cluster/ca.crt");
const namespace = process.env.DEMO_NAMESPACE || "openclaw";

function api(path) {
  return new Promise((resolve, reject) => {
    const req = https.get({
      hostname: "kubernetes.default.svc.cluster.local",
      path,
      ca,
      headers: { Authorization: `Bearer ${token}` },
    }, (res) => {
      let body = "";
      res.on("data", (chunk) => body += chunk);
      res.on("end", () => {
        if (res.statusCode !== 200) reject(new Error(`Kubernetes API ${res.statusCode}: ${body}`));
        else resolve(JSON.parse(body));
      });
    });
    req.on("error", reject);
  });
}

function podHealth(pod) {
  const phase = pod.status?.phase || "Unknown";
  const statuses = pod.status?.containerStatuses || [];
  const waiting = statuses.find((s) => s.state?.waiting);
  const terminated = statuses.find((s) => s.state?.terminated && s.state.terminated.reason !== "Completed");
  const restarts = statuses.reduce((n, s) => n + (s.restartCount || 0), 0);
  const ready = statuses.length ? statuses.every((s) => s.ready) : phase === "Succeeded";
  const bad = phase !== "Running" && phase !== "Succeeded" || waiting || terminated || !ready;
  return {
    name: pod.metadata.name,
    phase,
    ready,
    restarts,
    status: bad ? "needs-attention" : "healthy",
    reason: waiting?.state.waiting.reason || terminated?.state.terminated.reason || (ready ? "Ready" : "NotReady"),
  };
}

function html(rows) {
  const bad = rows.filter((r) => r.status !== "healthy");
  const items = rows.map((r) => `<tr><td>${r.name}</td><td>${r.phase}</td><td>${r.ready ? "yes" : "no"}</td><td>${r.restarts}</td><td class="${r.status}">${r.reason}</td></tr>`).join("");
  return `<!doctype html><title>Pod Doctor</title>
  <style>
    body{font-family:system-ui;margin:32px;background:#0a0c10;color:#eef2f7}
    h1{margin:0 0 8px} .muted{color:#8a93a3} .ok{color:#a3e635}.bad,.needs-attention{color:#ff6b6b}
    table{border-collapse:collapse;width:100%;margin-top:20px}td,th{border-bottom:1px solid #222a37;padding:10px;text-align:left}
    code{background:#121620;padding:2px 6px;border-radius:5px}
  </style>
  <h1>Pod Doctor</h1>
  <p class="muted">Namespace <code>${namespace}</code> · read-only Kubernetes token · refreshed on reload</p>
  <h2 class="${bad.length ? "bad" : "ok"}">${bad.length ? `${bad.length} pod(s) need attention` : "All watched pods look healthy"}</h2>
  <table><thead><tr><th>Pod</th><th>Phase</th><th>Ready</th><th>Restarts</th><th>Reason</th></tr></thead><tbody>${items}</tbody></table>
  <p class="muted">JSON: <a href="/pods.json">/pods.json</a></p>`;
}

http.createServer(async (req, res) => {
  try {
    if (req.url === "/healthz") return res.end("ok\n");
    const pods = await api(`/api/v1/namespaces/${namespace}/pods`);
    const rows = pods.items.map(podHealth);
    if (req.url === "/pods.json") {
      res.setHeader("content-type", "application/json");
      return res.end(JSON.stringify({ namespace, pods: rows }, null, 2));
    }
    res.setHeader("content-type", "text/html");
    res.end(html(rows));
  } catch (e) {
    res.statusCode = 500;
    res.end(String(e.stack || e));
  }
}).listen(8000, "0.0.0.0", () => console.log("Pod Doctor demo on :8000"));
JS
```

Check that it started:

```bash
oc -n openshell exec pod-doctor -c agent -- sh -c 'tail -5 /workspace/pod-doctor-demo.log'
oc -n openshell exec pod-doctor -c agent -- node -e 'require("http").get("http://127.0.0.1:8000/healthz", r => r.pipe(process.stdout))'
```

## Expose it

The lab shell has an `expose-agent` helper. Point it at your sandbox + the port your app serves on:

```bash
expose-agent pod-doctor 8000
```

Now open **port `8000`** from your Brev instance page — your copilot's endpoint, live in the browser. If Brev does not already show port `8000`, add it as a tunnel on the instance page and refresh.

<Callout type="note" title="Why not just a NodePort?">
A gateway-created sandbox is <strong>network-isolated by its supervisor</strong> — the agent's listening port lives in a private network namespace, so a plain NodePort to the pod can't see it. <code>expose-agent</code> uses OpenShell's own <code>openshell forward start</code>, which bridges the port <em>through the gateway + supervisor</em>. (Stop it with <code>openshell forward stop</code>; one forward at a time.)
</Callout>

## Make the demo pop

Create a tiny failure, reload the dashboard, then explain what the copilot is allowed to do:

```bash
oc -n openclaw delete deployment broken-demo --ignore-not-found
oc -n openclaw create deployment broken-demo --image=quay.io/this/image-does-not-exist:latest
```

The dashboard should show an unhealthy pod with an image-pull reason. In the OpenClaw UI, ask Pod Doctor:

> "Look at the unhealthy pod in `openclaw`. Explain the root cause and propose the exact `oc` command to fix it. Do not run writes."

Clean up when you are done:

```bash
oc -n openclaw delete deployment broken-demo --ignore-not-found
openshell forward stop
```

## Two surfaces to choose from

- **Your agent's own chat UI.** Expose the Pod Doctor OpenClaw control port (`expose-agent pod-doctor 18789`) and demo by chatting with your customized copilot — separate from the shared Shifty.
- **A custom task endpoint.** The richer demo: your copilot builds and serves the exact artifact your use-case needs — a live "cluster health" page, a remediation form, a metrics view (pair it with the [observability](/learn/monitoring) Prometheus endpoint).

<Callout type="rh" title="Why this lands">
A judge clicking your URL and watching your agent diagnose a real failing pod — on a real OpenShift cluster, behind a real policy guardrail — beats any slide. Build the endpoint, <code>expose-agent</code>, demo.
</Callout>

<Callout type="tip" title="Production note">
On a full cluster you'd use OpenShell's native service exposure (<code>ExposeService</code> → a per-sandbox loopback URL like <code>&lt;sandbox&gt;--&lt;service&gt;.apps.example.com</code>) or an OpenShift Route with its own hostname. This launchable uses <code>openshell forward</code> plus a Brev tunnel as the single-node shortcut.
</Callout>
