**OpenClaw** is the agent itself. It's deployed as an agent-sandbox `Sandbox`, which OpenShell reconciles into a pod running the OpenClaw **gateway** — a web *control UI* on port `18789` where you chat with and steer the agent.

## How it thinks: the model
OpenClaw ships no model. Its config declares a *provider* — base URL, API key, model id — using the OpenAI-compatible shape. We point it at a remote endpoint (e.g. an NVIDIA NIM). You'll configure this in **Part IV · Add a Model**.

## The agent's files — its memory & personality

OpenClaw isn't configured by a settings panel; it's shaped by **Markdown files in its workspace** (`~/.openclaw/workspace`). Each session the agent reads them, so they *are* its memory and character. Each file has a job:

- **`IDENTITY.md`** — *who* it is: `Name`, `Creature`, `Vibe`, `Emoji`, `Avatar`. (We seed Shifty's.)
- **`SOUL.md`** — *how* it behaves: core values + boundaries (be helpful, have opinions, confirm risky actions). The closest thing to an editable system prompt, in sections.
- **`BOOTSTRAP.md`** — the first-run kickoff (a “who am I?” interview). We replace it with a workshop welcome so the agent boots ready.
- **`AGENTS.md`** — house rules / capabilities the agent follows.
- **`TOOLS.md`** — the tools available to it.
- **`USER.md`** — facts about *you* (name, timezone, preferences).
- **`HEARTBEAT.md`** — what the agent does on its periodic check-in.

<Callout type="note" title="Seeded, not hard-coded">
We pre-seed <code>IDENTITY.md</code>/<code>SOUL.md</code>/<code>BOOTSTRAP.md</code> via the launchable's <code>workspace-seed</code> (an initContainer copies them, create-if-missing) so your agent boots as “Shifty 🦞”. You'll <em>read and edit</em> them hands-on in <strong>Part IV · Meet Shifty</strong>.
</Callout>

## Auth & pairing — what you'll do hands-on
- **Password** — the gateway runs `--auth password`; you type one known password.
- **Device pairing** — the first time a browser connects, the gateway asks a host operator to approve it (`openclaw devices list` / `approve`). You'll do this for real in **Part IV · Device Pairing**.
