Good news: **your launchable already did this.** When your Brev instance started, it ran the full provision for you (the `setup.sh` pipeline) — MicroShift, the OpenShell gateway, the OpenClaw agent, and the UIs are already coming up (or up). So you don't run anything heavy here; you just **confirm it's healthy**. **Open the lab shell** on the right (▶) — a real shell on your instance with `oc` ready.

## Confirm the cluster is up

```bash
oc get nodes
```

One `Ready` node = MicroShift is live. Now the workloads:

```bash
oc get pods -A | grep -vE "Running|Completed" || echo "✅ everything is Running"
```

If that prints `✅ everything is Running`, the whole stack is healthy. If a few are still pulling images on a fresh box, give it a minute:

```bash
# only if some pods aren't up yet — watch them settle, Ctrl-C when done
watch -n5 'oc get pods -A | grep -vE "Running|Completed" || echo ALL-RUNNING'
```

## Check the key pieces

```bash
oc -n openshell get pods         # OpenShell gateway (openshell-0) + agent-sandbox
oc -n openclaw  get pod openclaw-sandbox   # Shifty, your agent
```

When `openclaw-sandbox` is `Running`, your agent is alive.
