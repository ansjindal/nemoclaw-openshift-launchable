**OpenShell** (NVIDIA) is the piece that makes “run an AI agent safely” real. It's a **sandbox control plane**: agents don't run wherever they like — they ask OpenShell for a sandbox, and it creates an isolated, policy-bound pod.

## How it works

Click through the path an agent request takes to become a sandboxed pod:

<OpenShellArch />

In short: it **watches** the agent-sandbox CRD, **reconciles** each one into a policy-bound pod (deny-by-default), and **brokers** the sandbox's control surface and egress.

<Callout type="note" title="“NemoClaw's gateway” = the OpenShell gateway">
The OpenShell control plane (port 8080) is the platform's sandbox API — distinct from the OpenClaw agent's own control UI you open in a browser.
</Callout>

## On OpenShift
OpenShell installs from a Helm chart. The workshop scripts handle the OpenShift specifics: install the agent-sandbox CRD, grant the right SCCs to the OpenShell service accounts, and fix a few fsGroup/runAsUser defaults so it schedules cleanly on MicroShift.
