**OpenShell** (NVIDIA) is what makes “run an AI agent safely” real. It's not a single process — it's a **gateway** with several parts, sitting on top of an open-source Kubernetes primitive.

## The parts of OpenShell

<OpenShellArch />

The short version:

- **Gateway** — the control plane (gRPC :8080). Everything below lives here.
- **Compute driver** — creates and watches sandboxes (writes `Sandbox` CRs).
- **Inference router (L7 proxy)** — every sandbox calls one URL, `inference.local`; the gateway strips the sandbox's creds and injects the real backend key + model, so the agent **never holds the upstream key**.
- **Supervisor** (`openshell-sandbox`) — sideloaded into each sandbox to bootstrap and oversee the agent + enforce sandbox-side policy.
- **Policy + mTLS** — deny-by-default L7 rules (the schema in `policies/`) plus mTLS between components.
- **agent-sandbox** — the open-source [**kubernetes-sigs/agent-sandbox**](https://github.com/kubernetes-sigs/agent-sandbox) CRD + controller it all builds on.

## The relationship to agent-sandbox (the SIG project)

The bottom layer is community Kubernetes, not NVIDIA-specific: the `Sandbox` CRD (`agents.x-k8s.io`) and its controller come from **kubernetes-sigs/agent-sandbox**. OpenShell adds the gateway, the inference privacy router, the supervisor, and policy *on top* of that primitive. That's why `oc get sandboxes -A` works with plain `oc` — they're standard Kubernetes objects.
