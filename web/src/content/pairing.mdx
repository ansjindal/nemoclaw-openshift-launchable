When you first open the OpenClaw Control UI from your browser, it asks for **one-time device pairing** — a security feature: the gateway only trusts browsers a host operator has approved. Let's do it the real way, from the shell.

<Callout type="note" title="Why this exists">
OpenClaw gives every browser a device identity (a keypair). A remote browser must be approved once by someone with operator access — that's you, running the CLI <strong>inside your agent's sandbox</strong> via the OpenShell gateway.
</Callout>

## 1. See pending requests

When your browser shows “Device pairing required”, list the requests. **`sbox-openclaw`** is a lab-shell helper (there's no `openclaw` binary on this host) — it runs the OpenClaw CLI **inside your sandbox** via the gateway, i.e. it's shorthand for `openshell sandbox exec -n shifty -- openclaw …` (and adds the gateway URL + password for you):

```bash
sbox-openclaw devices list
```

You'll see a **Pending** row with a `Request` id (a UUID) for your browser, plus any already-`Paired` devices. If you already approved this browser, there may be no pending row — that's fine, continue.

## 2. Approve your browser

Copy the pending request id and approve it:

```bash
sbox-openclaw devices approve <REQUEST_ID>
```

<Callout type="tip" title="Approve the most recent">
In a hurry? <code>sbox-openclaw devices approve --latest</code> selects the newest pending request.
</Callout>

## 3. Reconnect

Back in the browser, reload — you're in. List devices again and your browser shows under **Paired**:

```bash
sbox-openclaw devices list
```

## Managing devices

```bash
sbox-openclaw devices remove <DEVICE_ID>   # revoke a device
sbox-openclaw devices clear                # clear all paired devices
```

<Callout type="rh" title="The real-world posture">
In production you'd keep pairing on and approve teammates deliberately. For a throwaway workshop box you could disable it (<code>gateway.controlUi.dangerouslyDisableDeviceAuth</code>), but learning the approve flow is the point here.
</Callout>
