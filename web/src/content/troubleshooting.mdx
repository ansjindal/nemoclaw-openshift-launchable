The gotchas we hit so you don't have to. Most are handled by the scripts — here's what to check if something's off (run the checks in the shell).

## Can't reach the UI on :30789 / :30900
NodePorts live inside the MINC container; a systemd forwarder publishes them on the host.

```bash
systemctl status minc-nodeport-forward.service
sudo ss -ltnp | grep -E ':30789|:30900'
sudo systemctl restart minc-nodeport-forward.service   # if you just recreated the agent
```

## OpenClaw asks for device pairing
That's expected — approve it (see **Part IV · Device Pairing**):

```bash
openclaw devices list
openclaw devices approve --latest
```

## Console shows “Please configure authentication”
The standalone console needs a static user token even with auth disabled — the scripts inject a cluster-admin SA token. If you see this, re-apply the console manifest.

## Console is empty / “no resources”
Not a bug — MicroShift lacks `project.openshift.io`. Use the **Project → All Projects** toggle or `/console/k8s/all-namespaces/pods`.

## Image pulls stall (“unexpected EOF”)
An MTU issue, not throttling — phase 20 clamps the node `eth0` MTU to 1400.

<Callout type="warn" title="podman, not docker">
If MicroShift won't come up, check <code>CONTAINER_ENGINE=podman</code> in <code>.env</code>. Docker's overlay breaks the nested CRI-O store.
</Callout>

<Callout type="note" title="Every phase is re-runnable">
Each <code>scripts/NN-*.sh</code> is idempotent — fix the cause and re-run just that phase.
</Callout>
