You've seen Shifty's sandbox and configured it. Now create a **brand-new sandbox from scratch** and watch the whole lifecycle — the same machinery the platform uses for every agent.

## The flow

<SandboxFlow />

In this lab we'll use the **direct CR path** (apply a `Sandbox` with `oc`). In a richer, multi-tenant setup a client would instead call the **OpenShell gateway** (gRPC :8080), which writes the same CR for you — but the reconcile → schedule → run steps are identical.

## 1. Declare a Sandbox

A `Sandbox` (`agents.x-k8s.io/v1beta1`) needs only a `podTemplate`. We reuse the OpenClaw image (already on the node) with a simple sleep command so there's no pull:

```bash
cat <<'EOF' | oc apply -f -
apiVersion: agents.x-k8s.io/v1beta1
kind: Sandbox
metadata:
  name: hello-sandbox
  namespace: openclaw
spec:
  shutdownPolicy: Delete
  podTemplate:
    spec:
      containers:
        - name: shell
          image: ghcr.io/openclaw/openclaw:2026.6.10
          imagePullPolicy: IfNotPresent
          command: ["sh", "-c", "echo ready; sleep 3600"]
EOF
```

## 2. Watch it reconcile

The **agent-sandbox controller** turns your CR into a pod. List sandboxes and watch the pod appear:

```bash
oc get sandboxes -A
oc -n openclaw get pod hello-sandbox -w
```

(Once it's `Running`, press Ctrl-C to stop watching.)

## 3. Run something inside it

It's a real, isolated sandbox — exec in:

```bash
oc -n openclaw exec hello-sandbox -- sh -c 'echo "hello from a brand-new sandbox"; hostname'
```

## 4. Customize it — give the sandbox an identity

A sandbox is yours to shape. You control its filesystem — write files into it:

```bash
oc -n openclaw exec hello-sandbox -- sh -c 'mkdir -p /workspace && cat > /workspace/IDENTITY.md <<MD
Name: Scout
Creature: a curious recon bot 🔭
Vibe: terse, precise, read-only
MD
cat /workspace/IDENTITY.md'
```

For a real **OpenClaw** agent, a file exactly like this in its workspace becomes its persona (the same `IDENTITY.md`/`SOUL.md` you met in *Meet Shifty*). So to spin a **custom agent**, you'd declare a Sandbox that (a) uses the OpenClaw image and (b) seeds its own `IDENTITY.md`/`SOUL.md` + config — the pattern the launchable uses for Shifty (`manifests/openclaw/workspace-seed/` + an initContainer):

```bash
# sketch: a customized agent = OpenClaw image + your own seeded identity/config
#   spec.podTemplate.spec.initContainers: copy your IDENTITY.md/SOUL.md into the workspace
#   spec.podTemplate.spec.containers[0]: ghcr.io/openclaw/openclaw + gateway args
```

<Callout type="tip" title="Your hackathon agents">
Each teammate can spin their <em>own</em> named agent with its own identity and policy — they're all just <code>Sandbox</code> CRs. Give one a recon personality, another a remediation one.
</Callout>

## 5. Tear it down

Because we set `shutdownPolicy: Delete`, removing the CR removes the pod:

```bash
oc -n openclaw delete sandbox hello-sandbox
oc get sandboxes -A
```

<Callout type="rh" title="This is the whole model">
Declare desired state (a CR) → a controller reconciles it into a pod → OpenShift enforces RBAC/SCC/NetworkPolicy → you use it → delete to clean up. Every agent, including Shifty, is just a <code>Sandbox</code> on this same path.
</Callout>

<Callout type="tip" title="Hackathon angle">
Your Cluster Copilot could <em>create sandboxes on demand</em> — spin a scratch sandbox to run a risky command, then delete it. The CR + controller pattern makes that a few lines of YAML.
</Callout>
